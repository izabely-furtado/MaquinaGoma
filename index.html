Diagrama do Padrão State Implementado

Explicação da Implantação do Método

Aplicando a teoria sobre state, nota-se no diagrama o encapsulamento de cada estado indicado em uma classe, e para a alteração o estado da classe contexto vamos ter uma ação invocada (+solicitaGoma(moeda : int)) de acordo com o fluxo indicado no exercício.

E como implementaremos isso?

Primeiro vamos ter que criar uma interface para os estados que será implementado pela classe MaquinaState como abstrata e sobreescrita por cada um dos estados.

  1. public interface IMaquinaState {
  2. void solicitaGoma(int moeda);
  3. }

Na classe Maquina a função +solicitaGoma de uma forma a qual solicita o pedido de acordo com o estado atual a qual se encontra a máquina, que está descrita da seguinte forma:

  1. public void solicitaGoma(int moeda) {
  2. this.meuEstado.solicitaGoma(moeda);
  3. System.out.println(this.meuEstado.toString());
  4. }

Já nos estados concretos, esta função é chamada de uma forma a qual processe o dado a partir de um determinado ponto e encaminhe para o próximo estado, da seguinte forma, na classe RecebeMoeda, por exemplo:

  1. @Override
  2. public void solicitaGoma(int moeda) {

  4. if (moeda == 1){
  5. System.out.println("Acione a Lavanca");
  6. this._maquina.setEstado(new GomaVendida(this._maquina));
  7. this._maquina.solicitaGoma(moeda);
  8. }
  9. else{
  10. System.out.println("Moeda inadequada inserida");
  11. this._maquina.setEstado(new SemMoeda(this._maquina));
  12. this._maquina.solicitaGoma(moeda);
  13. }
  14. }

Os pontos importantes que deve-se observar são:

  • A variável privada da classe Maquina;

  • Os construtores que recebem como parâmetro um objeto da classe Maquina.

A variável _maquina representar nossa classe contexto dentro do estado e vai ser utilizado para todas as alterações e consulta de valores da Maquina.

Os construtores recebem a própria classe contexto para a criação do estado.

Como todas as classes que vão representar um possível estado para nossa classe contexto tem um contrato com a interface IMaquinaState podemos criar uma variável na nossa classe a partir dessa interface para representar o estado.

Motivação

Em comparação a classe criada sem o padrão, temos uma classe menor e mais simples, uma vez que a decisão de qual estado ele vai estar depois da chamada dos métodos não cabe mais a classe contexto e sim a cada classe encapsulada que representa um estado do mesmo modo que a execução do método que esta sendo delegado ao estado atual. Devemos observar também que para setarmos qualquer estado estamos utilizamos “MeuEstado = new EstadoConcreto” isso ocorre tanto na criação da classe contexto quanto nas alterações dos estados.
